# Phase 4 ÔÇö Custom Types & Extended Encode Support

## Architecture Decisions

- **`SCHEMA` becomes `fn schema()`** ÔÇö enables compound types to build their schemas at runtime (OnceLock lazy-init) by concatenating sub-type schemas. Primitives return `&[op_byte]` trivially.
- **String tables remain compile-time `const STRING_TABLE: &'static [u8]`** ÔÇö generated by `#[derive(Encode)]` + `HasStringTable`. The `Metadata` struct carries a `string_tables_provider: fn() -> &'static [&'static [u8]]` that the macro fills lazily. The backend's `format_payload` receives the `[&[u8]]` list and resolves `st_idx:u8` + `st_off:u16 LE` offsets. 3 bytes per string reference, not 16.
- **Schema format for STRUCT/ENUM stays as documented** in `docs/schema.md`: `[opcode|n] [st_idx:u8] [(st_off:u16 LE + field_schema) ├Ś n]` for STRUCT; `[opcode|n] [vdx_k:u8] [st_idx:u8] [(st_off:u16 LE + var_schema) ├Ś n]` for ENUM.
- **`Box<T>`, `Arc<T>`, `Rc<T>` default to hot-path formatting** ÔÇö call `Display::to_string()` and encode as `&str` on the caller thread (one allocation, no ring buffer copy of T). Users who want zero-alloc copy-to-backend decode can explicitly `*deref()` their smart pointer to let the inner `T: Encode` impl take over.
- **Collections (`Vec`, `HashMap`, `BTreeMap`) serialize as sequences** ÔÇö `OP_SEQ` with element schemas. `HashMap`/`BTreeMap` are `OP_SEQ(OP_TUPLE(K, V))`. Conservative `max_encoded_size` with a user-configurable cap.

## Phase 4.0 ÔÇö `Encode::SCHEMA` Ôćĺ `fn schema()`

Replace `const SCHEMA: &'static [u8]` with `fn schema() -> &'static [u8]` in the `Encode` trait.

Required because stable Rust cannot concatenate `&[u8]` slices in `const` context. Compound types (structs with compound fields, enums with multi-field variants) need to build schemas that inline sub-type schemas. With `fn schema()`, the derive macro generates a `OnceLock<Vec<u8>>` that lazily builds the full schema by `extend_from_slice`-ing each sub-type's schema.

**Exploring `const`-time approach:** If `const { while }` blocks work on Rust 2024 edition, we prefer a `const{ [...] }` block over OnceLock to keep schemas truly compile-time. The proc macro emits:

```rust
fn schema() -> &'static [u8] {
    const RESULT: &[u8] = &{
        const PREFIX: &[u8] = &[0xA0 | 2, 0, /* st_offs... */];
        const F0: &[u8] = <Type0 as Encode>::schema();  // error: not const fn
        // ...
    };
    RESULT
}
```

Issue: `T::schema()` is `fn`, not `const fn`. We can't call it from `const` context. So `OnceLock` is the primary approach. Primitives override `schema()` to return a `const` slice (trivial).

## Phase 4.1 ÔÇö Thread String Tables Through Metadata Ôćĺ Backend

Add `string_tables_provider: fn() -> &'static [&'static [u8]]` to `Metadata`.

**Macro changes (`log_impl!`):**
- Add `static _STRING_TABLES: OnceLock<Box<[&'static [u8]]>> = OnceLock::new();` per call site
- On first successful log call, collect `HasStringTable::STRING_TABLE` for each arg
- For args without `HasStringTable`, push `&[]` as placeholder to maintain positional alignment
- Update `st_idx` in each schema to match the positional index of its string table

**Helper function:**
```rust
pub fn string_table_of<T: HasStringTable + ?Sized>(_: &T) -> &'static [u8] {
    <T as HasStringTable>::STRING_TABLE
}
```

**Backend changes (`format_body`, `format_payload`):**
- `format_body()` calls `metadata.string_tables_provider()` and passes result to `format_payload()`
- `format_payload()` for OP_STRUCT/OP_ENUM: uses the actual `string_tables: &[&[u8]]` slice instead of hardcoded `&[]`
- `decode_args()` also passes string tables through

## Phase 4.5 ÔÇö Built-in `Encode` for `Box<T>`, `Arc<T>`, `Rc<T>`

Default behavior: **hot-path format** ÔÇö call `Display::to_string()` on the caller thread, encode as `&str` into the ring buffer. One heap allocation (`to_string()`), no ring buffer copy of T.

Zero-alloc opt-in: user writes `log_info!(logger, "val={}", *box)` ÔÇö deref coerces to `&T`, which routes to `T: Encode` (copy T to ring buffer, backend decodes and formats).

```rust
impl<T: fmt::Display> Encode for Box<T> {
    fn schema() -> &'static [u8] { <&str as Encode>::schema() }
    fn encode_to(&self, buf: &mut [u8]) -> usize { self.to_string().as_str().encode_to(buf) }
    fn max_encoded_size(&self) -> usize { 512 }
}
```
