## Codebase Patterns

- Rust edition 2024 uses `extern crate` not needed; crate dependencies are auto-available.
- Module files go in `src/` with `pub mod` declarations in `lib.rs`.
- Sink trait uses `Send + Sync` bounds for thread-safe dispatch from backend.
- Logger and sinks use `Arc<dyn Sink>` for shared ownership.

## [2026-05-04] — Phase 2 (2.0–2.6): Zero-Allocation Hot Path
- Implemented LogArg trait with packed encoding (tags first, then all payloads) in src/arg.rs
- Supports i32, i64, u32, u64, usize, f32, f64, bool, &str, String with 1-byte tag + fixed/variable payload
- Added DisplayArg<T> and DebugArg<T> wrappers for fallback types (pre-format via Display/Debug)
- DecodedArg enum with as_display_string() + single generic decode_args() for backend
- format_with_args() replaces {} placeholders with decoded args
- Changed queue header from 4-byte to 8-byte (u32 length + u32 flags) for 8-byte alignment
- push() uses write_chunk_uninit + as_mut_slices + raw pointer zero-init for safety
- pop() now reads 8-byte headers and pads to 8-byte boundary
- Changed LogMessage from args_data: Vec<u8> to args_len: u16 (inline args follow header)
- serialize_header_into() writes raw bytes matching rkyv archive layout (32 bytes)
- decode() uses raw pointer cast instead of check_archived_root (avoids CheckBytes issues in edition 2024)
- Rewrote all 8 macros with new pattern: ($logger, $fmt:literal $(, $arg:expr)*)
- Hot path: compute sizes via log_max_size(), use 512-byte stack buffer or vec! fallback
- Encodes count byte + packed tags + sequential payloads into buffer
- Backend format_message() now decodes args from inline bytes via decode_args() then format_with_args()
- Updated integration tests for new macro signature (explicit arg syntax)
- Files: src/arg.rs (new), src/queue.rs, src/message.rs, src/macros.rs, src/backend.rs, src/lib.rs, tests/integration.rs, benches/logging.rs
- **Benchmark results:**
  - 1 integer: 121 ns → 10.4 ns (11.6×)
  - 2 floats: 249 ns → 11.5 ns (21.7×)
  - 3 strings: 237 ns → 15.3 ns (15.5×)
  - 1 Vec<String> (DebugArg): 316 ns → 204 ns (1.55×)
- **Learnings for future iterations:**
  - rkyv 0.7 lacks FallibleSerializer — had to use manual byte writing for header serialization
  - check_archived_root with CheckBytes derive has issues in edition 2024 — switched to raw pointer access
  - Declarative macros can't generate unique variable names — each arg's encoding must happen in separate {} blocks
  - rtrb's WriteChunkUninit lacks as_mut_ptr; used as_mut_slices + raw pointer zero-init pattern
  - The macro evaluates $arg multiple times — users must pre-bind rvalue args (e.g., DebugArg(foo())) to a local
  - Stack buffer approach (512 bytes) achieves zero-allocation for common case, vec! fallback for large messages
  - Packed tags-before-payloads layout proven efficient and simple to decode
  - 10 ns target almost met — remaining overhead is ring buffer copy (~2ns), level check (~1ns), encoding (~5ns)
